Documentación de API

Guía completa para integrar el sistema de predicción de vuelos

Base URL

{{ apiBaseUrl }}

POST /predict

Realiza una predicción de vuelo basada en los parámetros proporcionados. Retorna la previsión del vuelo y la probabilidad asociada.

Request Body

{{
`{
  "aerolinea": "string",           // Código IATA (ej: "AA", "IB", "LA")
  "origen": "string",              // Código IATA del aeropuerto (ej: "MAD", "GRU")
  "destino": "string",             // Código IATA del aeropuerto (ej: "MIA", "EZE")
  "fecha_Partida": "string",       // Formato: "yyyy-MM-dd HH:mm:ss" (ej: "2024-01-15 14:30:00")
  "distancia_km": number           // Distancia en kilómetros (máximo 7 dígitos enteros, 2 decimales)
}`}}

Ejemplo de Request

{{
`{
  "aerolinea": "AA",
  "origen": "SFO",
  "destino": "LAX",
  "fecha_Partida": "2024-01-15 14:30:00",
  "distancia_km": 559.23
}`}}

Response (200 OK)

{{
`{
  "prevision": "string",           // "A TIEMPO" o "RETRASADO"
  "probabilidad": number           // Valor entre 0 y 1 (ej: 0.85 = 85%)
}`}}

Ejemplo de Response

{{
`{
  "prevision": "A TIEMPO",
  "probabilidad": 0.85
}`}}

GET /predict/history

Obtiene el historial completo de todas las predicciones realizadas, ordenado por fecha de creación descendente (más recientes primero).

Parameters

No requiere parámetros.

Response (200 OK)

{{
`[
  {
    "id": number,                  // ID único de la predicción
    "aerolinea": "string",          // Código IATA de la aerolínea
    "origen": "string",             // Código IATA del aeropuerto de origen
    "destino": "string",            // Código IATA del aeropuerto de destino
    "fechaPartida": "string",       // Formato ISO 8601 (ej: "2024-01-15T14:30:00")
    "distanciaKm": number,         // Distancia en kilómetros
    "prevision": "string",          // "A TIEMPO" o "RETRASADO"
    "probabilidad": number,         // Valor entre 0 y 1
    "createdAt": "string",          // Fecha de creación (ISO 8601)
    "updatedAt": "string"           // Fecha de actualización (ISO 8601)
  }
]`}}

Manejo de Array Vacío

Si no se han realizado consultas hasta el momento, el endpoint retorna un array vacío [].

Headers

Todas las solicitudes deben incluir los siguientes headers:

{{
`Content-Type: application/json
Accept: application/json`}}

Códigos IATA

El sistema utiliza códigos IATA para aerolíneas y aeropuertos:

Aerolíneas Soportadas

  • • IB - Iberia
  • • G3 - Gol Transportes Aéreos
  • • AA - American Airlines
  • • LA - LATAM Airlines
  • • AR - Aerolíneas Argentinas

Aeropuertos Soportados

  • • MAD - Madrid
  • • GRU - São Paulo - Guarulhos
  • • AEP - Buenos Aires - Aeroparque Jorge Newbery
  • • MIA - Miami
  • • EZE - Buenos Aires - Ezeiza

Notas Importantes

  • • Formato de fecha: El campo fecha_Partida debe estar en formato yyyy-MM-dd HH:mm:ss
  • • Distancia: El campo distancia_km debe tener máximo 7 dígitos enteros y 2 decimales
  • • Códigos: Todos los códigos IATA deben estar en mayúsculas
  • • CORS: El backend debe estar configurado para aceptar peticiones desde el frontend
